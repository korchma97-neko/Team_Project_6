# Car Sorting Application (Java)
Консольное Java-приложение для управления и сортировки коллекции автомобилей. Проект демонстрирует применение классических шаблонов проектирования, работу с многопоточностью и создание кастомных структур данных.

## 📌 Описание проекта
Программа позволяет пользователю создавать, хранить, сортировать и анализировать коллекцию автомобилей. В основе проекта лежат следующие концепции:  
•	Шаблоны проектирования: Strategy, Decorator, Builder.  
•	Собственные коллекции: Реализация динамического списка.  
•	Многопоточность: Параллельный подсчет данных.  
•	Stream API: Современная обработка потоков данных.  
•	Manual Testing: Набор тестов для проверки логики.  
  
## 🚗 Модель автомобиля  
Каждый объект Car содержит следующие поля:  
•	Бренд (Brand)  
•	Модель (Model)  
•	Год выпуска (Year)   
  
Для гибкого создания объектов используется паттерн Builder.
Пример создания объекта:
```java
Car car = new Car.Builder()  
    .setPower(150)  
    .setModel("Toyota")  
    .setYear(2020)  
    .build();
```

## Используемые паттерны  
1. Strategy (Стратегия)  
Позволяет динамически менять алгоритм сортировки (Bubble Sort или Selection Sort) во время выполнения программы.  
•	Классы: SortStrategy, BubbleSortStrategy, SelectionSortStrategy, SortContext.  
2. Decorator (Декоратор)  
Добавляет дополнительную логику к существующим алгоритмам сортировки (например, фильтрация по четным/нечетным индексам) без изменения кода самих алгоритмов.  
•	Классы: SortDecorator, EvenOddDecorator.  
3. Builder (Строитель)  
Обеспечивает пошаговое создание сложного объекта Car, делая код более читаемым и защищенным от ошибок инициализации.  
  
## Многопоточность  
В приложении реализован алгоритм параллельного подсчета автомобилей определенного года выпуска.  
•	Коллекция разделяется на части.  
•	Каждая часть обрабатывается в отдельном потоке через MultiThreadCounter.  
•	Это значительно ускоряет обработку больших объемов данных.  
  
## Пользовательская коллекция: CustomCarList  
Вместо стандартных решений Java используется собственная реализация списка:  
•	Динамическое изменение размера при заполнении.  
•	Поддержка итераторов для использования в циклах.  
•	Интеграция с потоками (Stream API).
